Why WorldOrbit

Generic diagram tools can draw connections. WorldOrbit is built specifically for fictional orbital systems, Markdown-based lore repositories, codices, browser viewers, and content-first worldbuilding workflows.

Text first

Write diagrams like content, not like canvas state

WorldOrbit is designed for authors and worldbuilders who want to define systems in readable text, version them in Git, and drop them directly into Markdown-based projects.

Orbit aware

Made for stars, planets, moons, belts, structures, and phenomena

Instead of forcing orbital systems into generic node graphs, WorldOrbit speaks the language of stellar layouts, object metadata, viewpoints, annotations, and orbital presentation.

One source, many views

Render static diagrams, interactive viewers, or editor-driven scenes

The same source can drive a simple SVG diagram, a richer interactive view with tooltips and navigation, or the optional Studio editor for faster authoring.

Made for worldbuilding tools and content systems

WorldOrbit fits especially well into Markdown-native pipelines, worldbuilding CMS setups, lore encyclopedias, interactive setting documentation, and fictional atlas projects.

Simple workflow, scalable depth

Start with a tiny text block. Grow it into a richer, canonical schema document. Render it statically, explore it interactively, or open it in Studio when visual authoring becomes useful.

1

Write

Describe your stellar system in plain text. Keep it in Git, keep it in Markdown, and keep the source readable.

2

Render

Use the same document for a static diagram, an interactive view, or an atlas-style presentation with viewpoints and annotations.

3

Refine

Open the optional Studio editor when you want faster onboarding, visual placement handling, or side-by-side source and preview.

One format for writing, rendering, and optional visual editing.

WorldOrbit stays text-first at its core. The viewer and editor build on top of the same document model, so your source remains portable, inspectable, and useful across static pages, interactive atlases, and authoring tools.

Get started

Use WorldOrbit from npm, try it directly in the browser, or explore the repository examples and demos.

Install via npm

npm install worldorbit

Great for projects that want the parser, viewer, Markdown integration, or subpath exports like worldorbit/core and worldorbit/viewer.

Use in the browser

<script src="https://unpkg.com/worldorbit@2.5.4/dist/unpkg/worldorbit.min.js"></script>

Ideal for quick experiments, standalone demos, or embedding WorldOrbit in static and CMS-driven pages.
