First steps: how WorldOrbit documents are structured

A WorldOrbit document is plain text. You usually start with a schema line, then define a system, and then add objects like stars, planets, moons, belts, or structures.

Step 1

Start with the schema

For modern documents, begin with schema 2.0. This tells WorldOrbit which syntax style to expect.

Step 2

Define one system

A file normally contains one system. Think of it as the root container for your star system.

Step 3

Add objects

Then define stars, planets, moons, structures, and other elements. Objects can be simple at first and grow later.

Examples from simple to more complete

The easiest way to learn WorldOrbit is by building up complexity gradually. These examples are ordered from minimal to richer, atlas-style usage.

Example 1 — Minimal

A single star

This is the smallest useful idea: one system and one star.

WorldOrbit sourceminimal
schema 2.0

system Iyath
object star Iyath
Example 2 — Basic orbit

A planet orbiting a star

Now we add a planet and place it with orbit and semiMajor.

WorldOrbit sourcebasic
schema 2.0

system Iyath
object star Iyath
object planet Naar
  orbit Iyath
  semiMajor 1.18au
Example 3 — More orbital detail

Add orbital shape and orientation

Now the orbit becomes more descriptive with eccentricity, angle, inclination, and phase.

WorldOrbit sourceintermediate
schema 2.0

system Iyath
object star Iyath
object planet Naar
  orbit Iyath
  semiMajor 1.18au
  eccentricity 0.08
  angle 28deg
  inclination 24deg
  phase 42deg
Example 4 — Richer metadata

Use title, atmosphere, tags, and info

At this stage the object becomes more descriptive and useful for tooltips or atlas views.

WorldOrbit sourcemetadata
schema 2.0

system Iyath
object star Iyath

object planet Naar
  title "Naar"
  orbit Iyath
  semiMajor 1.18au
  atmosphere nitrogen-oxygen
  tags habitable homeworld
  info
    description "Homeworld of the Enari."
    climate "Temperate to dry"
Example 5 — Moon and structure

Add a moon and a special position

You can add more bodies and artificial objects such as stations or relays.

WorldOrbit sourcemulti-object
schema 2.0

system Iyath
object star Iyath

object planet Naar
  orbit Iyath
  semiMajor 1.18au

object moon Leth
  orbit Naar
  distance 220000km

object structure Relay-1
  at Naar:L4
  kind relay
Example 6 — Atlas-style document

Add defaults, viewpoints, and annotations

This is where WorldOrbit starts to feel like a richer viewer or atlas document, not just a raw object list.

WorldOrbit sourceatlas style
schema 2.0

system Iyath
  title "Iyath System"

defaults
  view isometric
  scale presentation
  preset atlas-card
  theme atlas

viewpoint overview
  label "Overview"
  summary "Fit the whole system."
  projection isometric

annotation naar-notes
  label "Naar Notes"
  target Naar
  body "Homeworld of the Enari."

object star Iyath
object planet Naar
  orbit Iyath
  semiMajor 1.18au

Beginner tips

These are the things that make learning WorldOrbit easier and prevent the most common mistakes.

Start small

Do not try to model your entire setting in the first file. Start with one system, one star, and one or two planets. Add more only after the basic structure works.

Use indentation consistently

Most WorldOrbit examples use two spaces for indented fields. Staying consistent keeps documents readable and easier to debug.

Think in layers

First define objects. Then add orbit details. Then add metadata. Finally add atlas-style features like defaults, viewpoints, and annotations.

Use the info block for free text

If something feels like lore text, description, notes, or long-form metadata, it usually belongs in info.

Quick checklist

  • Start with schema 2.0.
  • Define exactly one system per file.
  • Add objects like star, planet, and moon.
  • Use one placement mode per object: orbit, at, surface, or free.
  • Use units like au, km, d, y, and deg.

FAQ for beginners

A few short answers to the questions people often have when they first see the DSL.

Do I always need to write object?

No. Many examples can also work in shorter forms, but object type name is the clearer and more extensible style, especially in schema 2.0.

Do I need to know every parameter from the beginning?

No. You can start with only a star and a planet. WorldOrbit is designed to grow with your document.

What is the difference between orbit and at?

orbit describes a normal orbital relationship. at places something exactly at another object or a special point like a Lagrange point.

When should I use viewpoints and annotations?

Use them when the document should not only describe the system but also present it as a richer atlas or viewer experience.
