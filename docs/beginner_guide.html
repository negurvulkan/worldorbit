First steps: what a WorldOrbit document looks like

A WorldOrbit document is plain text. You usually start with a schema line, define a system, and then add objects. Objects can be simple at first and become richer later with orbital data, labels, notes, and presentation defaults.

1

Start with the schema

Use schema 2.0 for modern documents. It tells WorldOrbit which syntax version to parse.

2

Define one system

A file usually contains one system. Think of it as the root container for everything else.

3

Add objects and placement

Then add stars, planets, moons, structures, or belts and describe how they are positioned.

Examples from simple to advanced

These examples are deliberately staged. Start with the smallest possible diagram, then move on to a richer system, and finally look at a more complete representation of our solar system.

Example 1 — The smallest useful document

One system and one star. This is the simplest valid starting point.

schema 2.0

system Sol

object star Sun
Rendered result: a single star inside one system.

Example 2 — A small star system with planets and a moon

This adds orbit data, a moon, and a small station at a Lagrange point.

Sourceintermediate
schema 2.0

system Iyath
  title "Iyath System"

defaults
  view isometric
  scale presentation
  preset atlas-card
  theme atlas

object star Iyath

object planet Naar
  orbit Iyath
  semiMajor 1.18au
  eccentricity 0.08
  angle 28deg
  inclination 24deg
  phase 42deg
  atmosphere nitrogen-oxygen

object moon Leth
  orbit Naar
  distance 384000km
  phase 140deg

object structure Relay-One
  at Naar:L4
  kind relay
Rendered result: a system with a star, one planet, a moon, and a relay at a special orbital position.

Example 3 — A more complete solar system

This example tries to capture the structure of our solar system with the Sun, inner planets, gas giants, a moon, the asteroid belt, the rings of Saturn, and a few structures. It is intentionally richer so you can see how WorldOrbit scales from tiny diagrams to dense orbital maps.

Sourceadvanced
schema 2.0

system Sol
  title "Solar System"

defaults
  view topdown
  scale presentation
  preset atlas-card
  theme atlas

viewpoint overview
  label "Solar System Overview"
  summary "All major planets and selected structures."
  projection topdown

annotation inner-worlds
  label "Inner Worlds"
  target Earth
  body "Mercury, Venus, Earth, and Mars form the rocky inner system."

annotation outer-worlds
  label "Outer Giants"
  target Jupiter
  body "The outer system is dominated by gas and ice giants."

object star Sun
  radius 1sol
  mass 333000me

object planet Mercury
  orbit Sun
  semiMajor 0.39au
  phase 25deg
  class rocky

object planet Venus
  orbit Sun
  semiMajor 0.72au
  phase 75deg
  class rocky
  atmosphere carbon-dioxide

object planet Earth
  orbit Sun
  semiMajor 1au
  phase 135deg
  class rocky
  atmosphere nitrogen-oxygen

object moon Moon
  orbit Earth
  distance 384400km
  phase 220deg

object structure ISS
  orbit Earth
  distance 6771km
  kind station

object structure JWST
  at Earth:L2
  kind telescope

object planet Mars
  orbit Sun
  semiMajor 1.52au
  phase 210deg
  class rocky

object belt Main-Belt
  orbit Sun
  inner 2.1au
  outer 3.3au

object planet Jupiter
  orbit Sun
  semiMajor 5.20au
  phase 300deg
  class gas-giant

object moon Europa
  orbit Jupiter
  distance 671000km
  phase 40deg

object planet Saturn
  orbit Sun
  semiMajor 9.58au
  phase 20deg
  class gas-giant

object ring Saturn-Rings
  orbit Saturn
  inner 67000km
  outer 140000km

object planet Uranus
  orbit Sun
  semiMajor 19.2au
  phase 95deg
  class ice-giant

object planet Neptune
  orbit Sun
  semiMajor 30.05au
  phase 175deg
  class ice-giant

object structure Voyager-1
  free 160au
  kind probe

object phenomenon Heliosphere
  kind boundary
  source Sun
Rendered result: a dense, atlas-style view of the solar system with annotations, rings, belts, structures, and selected moons.

Beginner tips

These habits make WorldOrbit easier to learn and easier to maintain.

Start small

Define the system and one star first. Then add one planet at a time. It is easier to debug small steps than one huge document.

Use defaults later

Presentation settings like defaults, viewpoint, and annotation are useful, but you do not need them for your first diagram.

Think in layers

There is the system itself, the objects inside it, and optional presentation details. Keeping these layers in mind makes the syntax easier to understand.

What beginners usually need first
  • schema 2.0
  • system Name
  • object star Name
  • object planet Name with orbit and semiMajor
  • Optional: a moon, a belt, or a structure

FAQ

Short answers to the most common beginner questions.

Do I always need schema 2.0?

For modern documents, yes. It makes the source explicit and predictable.

Do I have to use every parameter?

No. WorldOrbit is flexible. Many diagrams start with just a system, a star, and one or two planets.

What is the difference between orbit and at?

orbit places an object on a normal orbit. at places it at a fixed reference position, such as a Lagrange point.

Can I use WorldOrbit in Markdown?

Yes. That is one of its core strengths. It is designed for Markdown-native worldbuilding and documentation workflows.
